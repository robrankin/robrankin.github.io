Packet Capture using tcpdump on Kubernetes Pods in Azure AKS

Rob Rankin published on 2022-04-21 included in tech
Assuming the target containers can actually install new software (apt install is available) what follows is a quick and very dirty method to run tcpdump on k8s/AKS containers in Azure. Install some needed utilities Use whatever pod label is required to target the right pods. kubectl get pods -l <LABEL> -o name | \ xargs -I{} kubectl exec {} -- apt-get -y update Install tcpdump, screen, psmisc, and rclone kubectl get pods -l <LABEL> -o name | \ xargs -I{} kubectl exec {} -- \ apt-get -y install tcpdump screen psmisc rclone Check kubectl get pods -l <LABEL> -o name | \ xargs -I{} kubectl exec {} -- tcpdump --version Start tcpdump in a screen Use whatever tcpdump <FILTER> here as needed.
